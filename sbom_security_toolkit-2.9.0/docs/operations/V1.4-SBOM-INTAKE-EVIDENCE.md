# v1.4 SBOM Intake and Evidence Workflows

This release turns the toolkit into a one-command SBOM operations workbench.

## One-command analysis

```bash
make analyze PROJECT=./my-app
```

The analyzer detects common ecosystems, finds or generates an SBOM when `syft` is available, then runs quality scoring, minimum-elements checks, policy gates, supplier intake, supplier questions, reports, and a local static dashboard.

## Supplier and vendor intake

```bash
make supplier-intake SBOM=vendor-bom.json
make supplier-questions SBOM=vendor-bom.json
make sbom-minimum-elements SBOM=vendor-bom.json
```

Outputs include a Markdown intake report, JSON evidence, a CSV questionnaire, and a ready-to-edit follow-up email.

## Exploitability evidence

```bash
make exploitability-record CVE=CVE-2099-0001 COMPONENT=pkg:pypi/example@1.0.0 STATE=under_investigation
make validate-edr EDR=exploitability-records/CVE-2099-0001-pkg-pypi-example-1.0.0.md
```

Exploitability Decision Records are human-review artifacts that can support later VEX decisions. Do not use them to auto-generate `not_affected` VEX without evidence.

## Privacy and safe sharing

```bash
make redact-sbom SBOM=internal-bom.json
```

Redaction removes URLs, emails, common local paths, and can hash internal-looking component names while preserving useful structure for analysis.

## Watch mode

```bash
make watch-sbom SBOM=bom.json VULNS=scan-results.json
```

Watch mode stores local state in `.sbom-watch/` and writes a delta report showing new/resolved vulnerabilities and component changes.

## Signed artifacts and release evidence

```bash
make checksums ARTIFACT_DIR=dist
make sign-artifacts ARTIFACT_DIR=dist
make verify-artifacts ARTIFACT_DIR=dist
```

The signing workflow creates checksums by default and uses `cosign` when installed. It never requires network signing tools for the demo path.

## CLI wrapper

```bash
python3 -m sbomops.cli analyze ./my-app
python3 -m sbomops.cli score bom.json
python3 -m sbomops.cli supplier-questions vendor-bom.json
```

The `sst` Make target prints CLI help. Packaging as a console script can be added later.
