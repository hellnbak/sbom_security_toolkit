# AI-Assisted Fuzzing

This module adds review-based AI workflows for SBOM fuzzing. It is intentionally conservative:

- AI output is advisory.
- Generated seeds, harnesses, tests, and campaigns go to review queues.
- Nothing generated by AI is executed or promoted automatically.
- Network model providers are opt-in.
- Prompt-only mode works without API keys or local models.

## Providers

Default mode writes prompts and deterministic draft artifacts only:

```bash
make ai-fuzz-seeds FORMAT=cyclonedx SCENARIO=dependency-cycles
```

Optional local Ollama mode:

```bash
AI_FUZZ_PROVIDER=ollama AI_FUZZ_MODEL=llama3.1 make ai-crash-triage CRASH=fuzzing/findings/example
```

Optional OpenAI-compatible endpoint:

```bash
AI_FUZZ_PROVIDER=openai-compatible \
OPENAI_COMPATIBLE_BASE_URL=http://127.0.0.1:8000/v1 \
AI_FUZZ_MODEL=local-model \
make ai-fuzz-campaign GOAL=sbom-parser-hardening
```

Optional GLM local/OpenAI-compatible profile:

```bash
AI_FUZZ_PROVIDER=glm GLM_BASE_URL=http://127.0.0.1:8000/v1 GLM_MODEL=glm-5.2 make ai-fuzz-seeds FORMAT=cyclonedx SCENARIO=dependency-cycles
```

Provider smoke test:

```bash
make ai-provider-test AI_PROVIDER=glm AI_MODEL=glm-5.2
```

See `docs/integrations/GLM-LOCAL-MODELS.md` for GLM setup patterns and safety guidance.

## Review queues

Artifacts are written under `ai_fuzz/review/incoming/`.

```bash
make ai-review-list
make ai-review-accept ITEM=<folder-name>
make ai-review-reject ITEM=<folder-name>
```

Accepted artifacts are still not automatically committed or executed. Review them before moving seeds into the regression corpus or enabling generated harnesses.
