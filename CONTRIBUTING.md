# Contributing

Thanks for helping improve SBOM Security Toolkit.

## Development Guidelines

- Keep the toolkit generic. Do not add customer names, internal hostnames, proprietary package names, real SBOMs, credentials, or private vulnerability data.
- Prefer safe defaults: localhost-only services, explicit environment variables for secrets, pinned versions for security tooling, and clear warnings for destructive or networked operations.
- Run syntax checks before submitting changes.
- Generated AI output must be reviewed before it is committed.
