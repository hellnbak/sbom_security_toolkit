# v2.14.1 Reconciliation Notes

## Why this release exists

The available source inputs had diverged:

- The attached package identified itself as v2.8.2 and contained the Quick Start/upload/automatic-report work.
- The public repository exposed later v2.9 through v2.13 capabilities and documentation, while its README contained conflicting current-version statements.
- The prior v2.14 runtime overlay implemented the intended reporting, wizard, and demo behavior but was not safe to apply to every older tree; applying it directly to the attached package produced an indentation defect and incomplete route integration.

v2.14.1 resolves those conflicts as a complete source tree rather than another overlay.

## Reconciliation rules

1. Preserve the attached package's established SBOM, fuzzing, Workbench, integration, and automatic-report foundations.
2. Preserve the later repository capability surface for release assurance, risk exceptions, provenance, release evidence, organizational context, connectors, guided workflows, and user-experience controls.
3. Replace overlay-only runtime behavior with source-integrated modules and tests.
4. Keep deterministic scan state separate from advisory narrative-report state.
5. Make demo and wizard behavior executable and observable.
6. Remove stale, duplicated, and contradictory release documentation.
7. Avoid committing generated evidence, runtime state, secrets, or local upgrade backups.

## Source integrity note

This package is a functionally reconciled source release built from the attached v2.8.2 source tree, the verified public repository interfaces and release history through v2.13, and corrected v2.14 runtime changes. It should be reviewed as a new v2.14.1 release rather than described as byte-for-byte identical to a prior Git tag.
