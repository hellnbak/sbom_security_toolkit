# v2.14.2 GitHub Reconciliation Hotfix

## Purpose

v2.14.2 repairs release packaging and upgrade defects discovered while applying v2.14.1 to the current GitHub repository. It retains the automatic-reporting, executable-wizard, and live-demo improvements while restoring the repository capability surface introduced in v2.9 through v2.13.

## Corrected defects

### Legacy editable installation

A compatibility `setup.py` now complements `pyproject.toml`. macOS environments starting with pip 21.2.x can install the project in editable mode, and the documented path upgrades pip, setuptools, and wheel inside the virtual environment before installation.

### Dependency error handling

YAML input is no longer passed to the JSON parser when PyYAML is unavailable. The loader emits a direct instruction to install the project dependencies.

### Non-destructive repository updates

`scripts/apply-release-safe.sh` copies release files without deleting destination-only files. It preserves Git metadata, virtual environments, runtime jobs, uploads, demo state, reports, projects, and local evidence. It also records the target's pre-application Git status and binary diff.

### Preflight after demo execution

Release preflight now distinguishes generated local runtime output from tracked or packaged release content. Running `make demo-live` no longer causes a false failure, while tracked or manifested runtime evidence still blocks a release.

### Restored repository capabilities

The release restores and validates CI workflows, release-assurance templates, connector examples, organization examples, policies, schemas, guided-experience documents, UX documents, and historical regression tests that the v2.14.1 mirror operation could remove.

## Validation expectations

```bash
make reconciled-test
make preflight-release
make demo-live
make preflight-release
```

The complete Python regression suite, rather than only the v2.14-specific tests, is part of `make reconciled-test`.
