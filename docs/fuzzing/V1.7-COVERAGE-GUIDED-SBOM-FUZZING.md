# v1.7 Coverage-Guided SBOM Fuzzing Lab

This release expands fuzzing from basic harnesses into a local fuzzing lab for SBOM tools and workflows.

## What was added

- Schema-aware CycloneDX, SPDX, VEX, and Dependency-Track payload seed generators.
- AFL++ scaffold with dictionary and custom-mutator bridge support.
- Toolchain fuzzing for local Syft, Trivy, Grype, and OSV-Scanner installations.
- Stateful local-only Dependency-Track workflow fuzzing scaffold.
- Scanner metamorphic testing to compare scanner behavior across harmless SBOM transformations.
- Fuzzing dictionaries and budget profiles.
- Bug-class campaigns for parser DoS, silent component drops, scanner disagreement, VEX logic errors, and license policy bypasses.
- Advisory draft generation for reproducible fuzz findings.
- Fuzzing status report and all-local fuzzing command.

## Safety model

All networked or API-oriented fuzzing defaults to localhost/private targets. AI-assisted workflows generate prompts, suggestions, and review items; they do not automatically execute generated code or promote generated seeds.

## Useful commands

```bash
make fuzz-generate-cyclonedx EDGE=dependency-cycle COUNT=25
make fuzz-toolchain SBOM=test-sboms/clean/minimal-cyclonedx.json
make fuzz-stateful-dtrack
make fuzz-metamorphic-scanners SBOM=test-sboms/clean/minimal-cyclonedx.json
make fuzz-all-local
```
