# AI-Assisted Fuzzing Workflows

SBOM Security Toolkit v1.6 adds AI-assisted fuzzing workflows. The goal is not to let a model run fuzzing by itself. The goal is to use AI to propose better seeds, mutation plans, semantic oracles, crash explanations, regression tests, harness drafts, coverage-gap ideas, campaign profiles, and scanner-disagreement explanations.

## Safety model

The implementation follows a strict review model:

- AI proposes; deterministic tools validate; humans approve.
- Prompt-only mode is the default and requires no model or network access.
- Optional providers are local Ollama or an OpenAI-compatible endpoint.
- Generated artifacts are written to `ai_fuzz/review/incoming/`.
- Seeds generated by the deterministic fallback are written to `fuzzing/corpus/ai/incoming/`.
- Nothing is promoted into the regression corpus automatically.
- Generated harnesses and tests are drafts only.
- Redaction is applied to prompt excerpts before they are sent to a configured provider.

## Commands

```bash
make ai-fuzz-seeds FORMAT=cyclonedx SCENARIO=dependency-cycles
make ai-mutation-plan SBOM=test-sboms/clean/minimal-cyclonedx.json
make ai-oracle-suggest TARGET=sbomops/minimum_elements.py
make ai-crash-triage CRASH=fuzzing/findings/example
make ai-regression-test CRASH=fuzzing/findings/example
make ai-fuzz-harness TARGET=sbomops/redact.py
make ai-coverage-suggest COVERAGE=fuzzing/reports/fuzz-coverage.md
make ai-fuzz-campaign GOAL=dependency-track-import-hardening
make ai-explain-disagreement REPORT=reports/scanner-compare/scanner-comparison.json
make ai-review-list
```

## Provider configuration

Default prompt-only mode:

```bash
make ai-fuzz-seeds FORMAT=cyclonedx SCENARIO=vex-contradictions
```

Ollama:

```bash
AI_FUZZ_PROVIDER=ollama AI_FUZZ_MODEL=llama3.1 make ai-fuzz-campaign GOAL=supplier-sbom-hardening
```

OpenAI-compatible local endpoint:

```bash
AI_FUZZ_PROVIDER=openai-compatible \
OPENAI_COMPATIBLE_BASE_URL=http://127.0.0.1:8000/v1 \
AI_FUZZ_MODEL=local-model \
make ai-crash-triage CRASH=fuzzing/findings/example
```

## Recommended workflow

1. Generate AI-assisted seeds or campaign drafts.
2. Review `ai_fuzz/review/incoming/`.
3. Accept or reject the review item.
4. Manually promote useful seeds into `fuzzing/regression/corpus/`.
5. Run `make fuzz-regression`.
6. Commit only reviewed artifacts.
