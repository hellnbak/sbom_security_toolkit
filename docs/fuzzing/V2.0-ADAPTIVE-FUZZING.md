# v2.0 Adaptive SBOM Fuzzing

This release turns fuzzing into a closed-loop, local-first workflow:

1. collect findings and corpus metadata in a local SQLite fuzzing knowledge base,
2. recommend the next campaign with `make fuzz-plan`,
3. run benchmark and scanner compatibility suites,
4. exercise curated truth-set SBOMs,
5. create replay packs for findings,
6. use AI-assisted harness loops and provider evaluations safely,
7. optionally run ClusterFuzzLite PR/nightly workflows.

The toolkit intentionally does **not** automatically execute AI-generated code, promote AI-generated seeds, or make final VEX/exploitability decisions.

## Commands

```bash
make fuzz-kb-init
make fuzz-plan
make ai-harness-loop TARGET=sbomops/minimum_elements.py
make ai-fuzz-loop GOAL=spdx-parser-hardening AI_PROVIDER=glm
make fuzz-benchmark
make sbom-tool-compatibility
make scanner-truthset
make fuzz-replay-pack FINDING=test-sboms/truthset/log4j-direct.json
make ai-fuzz-eval AI_PROVIDERS="none,glm"
```

## Research report generation

Research-report generation is intentionally not included in this release. The toolkit can create benchmark, compatibility, truth-set, and replay artifacts that can be reviewed manually before any public write-up.
