# v2.1 Intelligent Fuzzing Operations

v2.1 turns the Fuzzing Lab into a more connected local fuzzing operations workflow.

## New capabilities

- Fuzzing intelligence scoring ranks inputs, findings, and corpus entries by usefulness.
- Corpus promotion recommendations classify inputs as regression candidates, nightly seeds, duplicates, or review items.
- Harness quality audit checks generated or hand-written harnesses for unsafe behavior and weak target coverage.
- AI harness quality loop drafts, audits, and prepares repair prompts without auto-executing generated code.
- AI seed-generator synthesis creates reviewed generator drafts instead of only one-off seeds.
- Grammar-mutator scaffold produces structured samples for CycloneDX, SPDX, purl, license, and VEX formats.
- Method-targeted coverage hints help identify missing branch/seed classes for specific Python functions.
- Semantic format-diff checks compare normalized meaning across SBOM inputs.
- Vulnerability matching fuzzing generates identity ambiguity cases for scanners.
- VEX logic fuzzing generates contradiction and overclaim scenarios.
- Evil supplier scenarios test supplier intake against valid-looking but problematic SBOMs.
- AI fuzz red-team checks evaluate whether a model follows the toolkit's review-gated safety rules.
- ClusterFuzzLite result import and CI dashboard helpers make CI output visible locally.
- Finding lifecycle tracking moves fuzz findings through found, triaged, minimized, regression-added, fixed, verified, and archived states.

## Web UI

The local workbench Fuzzing Lab now exposes these workflows through the browser and adds a Fuzzing Dashboard page for intelligence, compatibility, AI red-team, and lifecycle artifacts.

## Important safety rule

AI proposes. Deterministic tooling validates. Humans approve.
